Question #1 : Explain what does it mean Immutable infrastructure?
Immutable infrastructure refers to an infrastructure setup where servers or environments are not modified after they are deployed. Instead of updating or patching an existing server, the entire server or container is replaced with a new version. This ensures consistency and prevents configuration drift. Immutable infrastructure helps to improve the reliability and scalability of systems, as every change is explicitly planned and deployed as a new instance, rather than being patched or updated on the fly.

Question #2 : Explain the next command:
$docker exec -it Ubuntu /bin/bash
The command $docker exec -it Ubuntu /bin/bash is used to start an interactive shell session within a running Docker container. Here's the breakdown of the command:

  • docker exec: Executes a command in a running container.
  • -it: Combines two options:
    • -i: Interactive mode, keeps STDIN open.
    • -t: Allocates a pseudo-TTY, allowing for an interactive terminal.
  • Ubuntu: The name or ID of the running container.
  • /bin/bash: The command to execute inside the container, providing a Bash shell to interact with the containerâ€™s file system.
